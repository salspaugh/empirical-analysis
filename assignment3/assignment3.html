def increment_timing_bucket(bins, time):
    bins[int(time)] += 1
    return bins

def bin_tweets_per_day(timings):
    timings.sort()
    duration_days = (timings[-1] - timings[0])/60./60./24.
    timings = [(t - timings[0])/60./60./24. for t in timings]
    bins = range(int(duration_days)+1)
    ntimings = { b:0 for b in bins }
    reduce(increment_timing_bucket, timings, ntimings)
    frequencies = [count for (bucket, count) in sorted(ntimings.items())]
    return bins, frequencies

def get_interarrivals(times):
    times.sort()
    interarrivals = []
    for idx, curr in enumerate(times[:-1]):
        next = times[idx+1]
        interarrivals.append(next - curr)
    return interarrivals

Because this data is from six years prior, I would not expect the percent of bots to be the same in it as in 2014, but I would expect it to be somewhat close, because bots were most likely already a thing in 2008, so I wouldn't expect a big surge in proportion later.

Summary

Because of the bizarre sampling that was done to collect this data, I don't really feel that we can draw certain conclusions from it. But I tried anyway.

  1. There are 519 users represented in the data set.
  2. Something is fishy about the sampling process used to collect these tweets. The number of timestamps per user follows a very strange pattern, with large numbers of users having exactly 30 tweets, or exactly 500 tweets, or exactly some number in-between. I would have assumed there would be more variation and randomness in this distribution. This indicates that the way these tweets were sampled is weird.
  3. There are about 134K tweets in the data set. Due to the weird sampling problem, this may not be meaningful.
  4. Tweets occur over a period of about two years starting in mid-2008. Again due to the weird sampling problem, it's unclear if the duration is meaningful.
  5. The number of tweets per day spikes twice in the last half of the second year. Again due to weird sampling issues, this is hard to interpret. Aside from that, the arrival patterns don't reveal much.
  6. About 30 users have extremely suspiciously high tweet rates. This is about 5% of users.
  7. There are suspicious spikes in the interarrival distributions at multiples of 60 seconds, possibly due to bots.
  8. The interarrival distribution is very skewed. This is especially so compared to the arrival distribution.
  9. The users with both skewed interarrivals and high tweet rates may be bots.
  10. There may be about 2.5-8% bots in the data set. This compares favorably with estimates from Twitter dating to approximately August 2014.

Here are some things I should have tried looking at, but didn't due to lack of time: * user arrivals * fit to known distributions * Q-Q plots * user names * Fourier analysis of arrival or interarrival times

In terms of metadata, I obviously wish that there was information about how and why these tweet times were collected, because I think it's really weird to have these huge plateaus in the number of tweets per user. This complicates drawing conclusions.
